    用hexo和guthub建立博客

    发表于 2022-06-02 更新于 2022-06-03

    安装所需

    下载安装node.js https://nodejs.org/en/
    建议下载LTS 区别：LTS–长期支持版，稳定 current–最新版
    一直下一步无脑安装 在cmd输入 node -v 检测安装是否成功 成功如下
    node -v
    下载安装git https://git-scm.com/
    一直下一步无脑安装 在cmd输入 git –version 检测安装是否成功 成功如下
    git --version
    安装hexo 使用npm命令安装hexo 命令如下
    npm install hexo-cli -g
    卸载hexo 命令
    npm uninstall hexo-cli -g
    检查是否安成功及版本 输入命令hexo -v

    建立一个GitHub仓库及获取ssh Keys

    如有网络问题可以使用科学上网或加速器

    点击new新建一个仓库
    输入仓库名字github用户名为前缀.github.io其他不要更改

    勾选public然后点击创立/Create repository

    获取ssr Keys

    在桌面右击鼠标进入git bash here

    输入
    ssh-keygen -t ras -C “输入GitHub的邮箱地址” 注意空格

    然后回车四次会产生一个文件.ssh 密钥文件

    在C:\Users\用户名.ssh 中找到id_rsa.pub用编辑器打开然后复制ssh

    进入GitHub——>settngs->SSH and GPG keys->new key
    依次填写title key(把id_rsa.pub文件中的ssh复制粘贴到里面) 然后点击add ssh key
    检测SSh是否绑定成功

    在git bash here输入ssh -T git@github.com
    $ ssh -T git@github.com
    成功如下

    在本地生产博客

    在桌面建立博客/blog文件夹然后空白区域右键git bash here输入命令hexo init初始化hexo，就会在本文件夹生产hexo所需的文件
    hexo init
{%asset_img seventeen.png%}
    hexo s—-生成本地静态hexo的页面
    hexo s

    在网页打开 http://localhost:4000/ 静态hexo的页面不能关闭这个页面，关闭后打不开静态hexo的页面，像服务器

    将博客发布到网上

    打开博客/hexo文件夹找到**_config.yml**最后添加/修改

    deploy:
        type: git "这个统一的"
        repository: https://github.com/wsmyyzmkta/wsmyyzmkta.github.io.git "这是仓库地址每个人不一样"
        branch: main "这个统一的"
    在博客/hexo根目录右键git bash here，输入命令 安装hexo-deployer-git自动部署发布工具：npm install hexo-deployer-git –save
    npm install hexo-deployer-git --save

    这里如果出错了，不要慌，删除电脑user/用户名/下的隐藏文件 .npmrc（注意是隐藏文件）再次输入代码即可
    输入hexo g—-生成页面的一个工具
    hexo g
    输入hexo d—-把本地文件上传GitHub仓库
    hexo d
    在hexo****d过程中会弹框先输入用户名 第二个弹窗再输入GitHub密码
    注意第二个弹框根据GitHub更新变动，前时是令牌，现在是Github授权
    这是你会发现我们建立的GitHub仓库有了代码hexo代码

    现在可以用仓库名字访问网上而不是本地hexo博客了

    如不知道可以在仓库设置的GitHub Pages查找我们的网站

    重新上传请用hexo c hexo g hexo d

    hexo c 清理缓存
hexo g 生成静态文件
hexo d 来将文档部署到Github上
