CSS grid网格布局

发表于 2022-07-17

css grid网格布局 横行竖列

在父元素声明grid布局

在父元素声明grid的布局
display：grid或者
display：inline-grid
让其下的子元素成为网格项目

grid-template-columns grid-template-rows repeat()

grid-template-columns: 200px 200px 200px;

grid-template-columns: 200px 200px 200px 200px 200px;

属性设置列的宽度 同时还是设置有多少列
grid-template-rows: 100px 100px ;

属性设置行的高度 同时还是设置有多少行
repeat()
repeat 重复 /rəˈpēt/
rapeat()接受两个参数
第一个参数时rereap的次数
第二个时重复的值
grid-template-columns: 200px 200px 200px 200px 200px;
用repeat()函数：
grid-template-columns:repeat(4,200px);
重复了4次，重复的值是200px；不0

auto-fill

grid-template-columns:repeat(auto-fill ,200px);

自动填充，只要浏览器能够容纳得下，就可以放置元素，

grid-gap

grid-gap：20px;

同时设置行列的间距
grid-row-gap：20px;
设置row的间距
grid-columns-gap：20px;
设置colunms的间距

fr

一个长度单位来帮助我们创建灵活的网格轨道
grid-template-columns: 200px 1fr 2fr;
表示第一个列宽设置为 200px，200px那个元素后面剩余的宽度分为两部分，宽度分别为剩余宽度的 1/3 和 2/3

minmax()

给网格元素一个最小和最大的尺寸,所以它接受两个参数，分别为最小值和最大值。
grid-template-columns: 1fr 1fr minmax(300px, 2fr)
意思是，第三个列宽最少也是要 300px，但是最大不能大于第一第二列宽的两倍。

auto

由浏览器决定长度。
grid-template-columns: 200px auto 200px
表示第一第三列为 100px，中间由浏览器决定长度，

grid-template-areas

属性用于定义区域，一个区域由一个或者多个单元格组成
一般这个属性跟网格元素的 grid-area 一起使用，我们在这里一起介绍。 grid-area 属性指定项目放在哪一个区域

grid-auto-flow

属性控制着自动布局算法怎样运作，精确指定在网格中被自动布局的元素怎样排列。默认的放置顺序是”先行后列”，即先填满第一行，再开始放入第二行，即下图英文数字的顺序 one,two,three…。这个顺序由 grid-auto-flow 属性决定，默认值是 row。
细心的同学可能发现了一个问题，就是第五个项目和第六个项目之间有个空白（如下图所示），这个是由于第六块的长度大于了空白处的长度，被挤到了下一行导致的。在实际应用中，我们可能想让下面长度合适的填满这个空白，这个时候可以设置 grid-auto-flow: row dense，表示尽可能填满表格。代码以及效果如下所示：

justify-items 属性、align-items 属性以及 place-items 属性

justify-items 属性设置单元格内容的水平位置（左中右），align-items 属性设置单元格的垂直位置（上中下）

下面以 justify-items 属性为例进行讲解，align-items 属性同理，只是方向为垂直方向。
.container {
justify-items: start | end | center | stretch;
align-items: start | end | center | stretch;
}

justify-content 属性、align-content 属性以及 place-content 属性

这是属性作用于页面位置
start - 对齐容器的起始边框
end - 对齐容器的结束边框
center - 容器内部居中
space-around - 每个项目两侧的间隔相等。所以，项目之间的间隔比项目与容器边框的间隔大一倍
space-between - 项目与项目的间隔相等，项目与容器边框之间没有间隔
space-evenly - 项目与项目的间隔相等，项目与容器边框之间也是同样长度的间隔
stretch - 项目大小没有指定时，拉伸占据整个网格容器

用hexo和guthub建立博客

发表于 2022-06-02 更新于 2022-06-03

安装所需

下载安装node.js https://nodejs.org/en/
建议下载LTS 区别：LTS–长期支持版，稳定 current–最新版
一直下一步无脑安装 在cmd输入 node -v 检测安装是否成功 成功如下
node -v
下载安装git https://git-scm.com/
一直下一步无脑安装 在cmd输入 git –version 检测安装是否成功 成功如下
git --version
安装hexo 使用npm命令安装hexo 命令如下
npm install hexo-cli -g
卸载hexo 命令
npm uninstall hexo-cli -g
检查是否安成功及版本 输入命令hexo -v

建立一个GitHub仓库及获取ssh Keys

如有网络问题可以使用科学上网或加速器

点击new新建一个仓库
输入仓库名字github用户名为前缀.github.io其他不要更改

勾选public然后点击创立/Create repository

获取ssr Keys

在桌面右击鼠标进入git bash here

输入
ssh-keygen -t ras -C “输入GitHub的邮箱地址” 注意空格

然后回车四次会产生一个文件.ssh 密钥文件

在C:\Users\用户名.ssh 中找到id_rsa.pub用编辑器打开然后复制ssh

进入GitHub——>settngs->SSH and GPG keys->new key
依次填写title key(把id_rsa.pub文件中的ssh复制粘贴到里面) 然后点击add ssh key
检测SSh是否绑定成功

在git bash here输入ssh -T git@github.com
$ ssh -T git@github.com
成功如下

在本地生产博客

在桌面建立博客/blog文件夹然后空白区域右键git bash here输入命令hexo init初始化hexo，就会在本文件夹生产hexo所需的文件
hexo init
{%asset_img seventeen.png%}
hexo s—-生成本地静态hexo的页面
hexo s

在网页打开 http://localhost:4000/ 静态hexo的页面不能关闭这个页面，关闭后打不开静态hexo的页面，像服务器

将博客发布到网上

打开博客/hexo文件夹找到**_config.yml**最后添加/修改

deploy:
        type: git "这个统一的"
        repository: https://github.com/wsmyyzmkta/wsmyyzmkta.github.io.git "这是仓库地址每个人不一样"
        branch: main "这个统一的"
在博客/hexo根目录右键git bash here，输入命令 安装hexo-deployer-git自动部署发布工具：npm install hexo-deployer-git –save
npm install hexo-deployer-git --save

这里如果出错了，不要慌，删除电脑user/用户名/下的隐藏文件 .npmrc（注意是隐藏文件）再次输入代码即可
输入hexo g—-生成页面的一个工具
hexo g
输入hexo d—-把本地文件上传GitHub仓库
hexo d
在hexo****d过程中会弹框先输入用户名 第二个弹窗再输入GitHub密码
注意第二个弹框根据GitHub更新变动，前时是令牌，现在是Github授权
这是你会发现我们建立的GitHub仓库有了代码hexo代码

现在可以用仓库名字访问网上而不是本地hexo博客了

如不知道可以在仓库设置的GitHub Pages查找我们的网站

重新上传请用hexo c hexo g hexo d

hexo c 清理缓存
hexo g 生成静态文件
hexo d 来将文档部署到Github上
